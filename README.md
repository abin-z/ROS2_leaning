# ROS2_leaning

简单来说，**ROS 2 是一个用于开发机器人系统的开源软件框架**。

你可以把它理解为机器人的“操作系统”，但它并不是像 Windows 或 Linux 那样直接管理硬件，而是提供了一套**通信、调度和工具**，让机器人的各个部件能够协同工作。

它是现代机器人开发的事实标准，被广泛应用于工业、自动驾驶、服务机器人等几乎所有领域，是进入机器人行业的重要技能。

为了帮你更好地理解，我们来拆解一下：

### 一、ROS 2 的核心：解决机器人开发中的通信问题

一个机器人（比如一个移动底盘）有很多传感器和部件：激光雷达、摄像头、电机、控制器等。ROS 2 的核心作用是让这些部件之间能够**高效、可靠地交换信息**。

它主要通过三种通信方式来实现，你可以对比成"微信群聊"、"打电话"和"下订单"：

| 通信方式           | 比喻       | 特点                                                         | 场景举例                                                     |
| :----------------- | :--------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **话题 (Topic)**   | 微信群聊   | **异步、持续性**。一方发布消息，所有订阅者都能收到。         | 摄像头持续发布“图像”消息，导航模块和显示界面都订阅这个“话题”来获取图像。 |
| **服务 (Service)** | 打电话     | **同步、一次性**。客户端发出请求，等待服务器回复。           | 节点A请求“计算路径”，节点B计算后回复“路径是[xxx]”。          |
| **动作 (Action)**  | 下外卖订单 | **长时间执行，可反馈**。发送目标，可中途取消，并持续收到进度反馈。 | 控制机械臂去抓取物体，可以一直收到“已移动到30%”的进度反馈。  |

> **一个形象的类比**：
> ROS 2 的通信机制很像**快递系统**。你可以把“传感器数据”或“控制指令”想象成一个个**快递包裹**。不同的通信方式，就是寄送这些包裹的不同方式。
>
> - **话题**就像**订阅报纸**：你订阅了，报纸（数据）就会定期送到你信箱，不用每次都打电话去要。
> - **服务**就像**发挂号信**：你寄出信后，会等待对方寄回一封回执，一来一回，当场办结。
> - **动作**就像**寄一个需要跟踪的快递**：你寄出后，会不断收到“已揽收”、“运输中”、“派送中”的进度通知，直到对方签收才算完成。

### 二、ROS 2 的组成：不仅是一套通信规则

除了通信，ROS 2 还提供了完整的开发生态：

1. **节点 (Node)**：一个可执行程序的最小单位。一个机器人可以有很多节点，比如一个“摄像头节点”、一个“激光雷达节点”和一个“控制节点”。
2. **包 (Package)**：ROS 2 软件的打包形式。你可以通过 `sudo apt install ros-humble-slam-toolbox` 这样的命令来安装别人写好的功能包（比如建图包、导航包）。
3. **分布式能力**：ROS 2 的节点可以不局限于同一台电脑。你可以让摄像头节点运行在树莓派上，而运行复杂算法的节点运行在性能更强的PC上，它们通过网络即可通信。
4. **丰富好用的工具**：
   - **`rqt`**：一个图形化调试工具集，可以查看所有节点和话题的实时通信状态。
   - **`rviz2`**：强大的3D可视化工具，可以显示机器人模型、激光雷达点云、摄像头图像等。
   - **`rosbag2`**：录制和回放工具。你可以先开车跑一圈，录制下所有传感器数据，然后回家在电脑上反复回放这些数据来调试算法，无需真车。

### 三、ROS 2 vs. ROS 1：为什么直接学新的ROS2？

ROS 1 在2010年左右发布，是开创性的，但也存在一些设计上的问题，比如**实时性差、分布式通信依赖一个中心网络节点（如果这个节点挂了整个系统就挂了）、对嵌入式支持不好**。

ROS 2 在2017年推出，完全重新设计，**解决了上述所有痛点**：

- 采用**DDS（数据分发服务）** 作为通信中间件，实现了真正的分布式、高可靠、支持实时通信。
- 去掉了**中心节点**，系统更健壮。
- 支持Windows、macOS，特别是对**嵌入式系统（如树莓派、STM32）** 的支持更好。

**官方已明确 ROS 1 将停止维护，所有新项目都应基于 ROS 2 开发。**
